Performance Log Analysis

Oct 2011

Performance log have execution time information for many entities. Its a time consuming effort to read log and find impacted area. Performance analysis done on local machine generally doesn't help. As DB latency and API performance varies grately in local machine from production. Before final performance testing which happens near UAT, we want to do it sooner so that issues can be fixed during development timeitself. Development phase is cheapest time to fix performance issue. To reduce manual effort of reading performance log this tool is created. It uses ordering stage and phase information to highlight performance issue. Best part is grouping of information which is achieved by html and javascript. And omoting very small value. Which helps in removing lots of cluster. On expanding information is shown in tabular form with minimal information. So that analysis of log is matter of second. 30 rules were created to identify which performance impact can be ignored. Like parent-child relation, 0 value etc. Anything deeper then that will require profiler for analysis. Plus a bifercation of time spent on DB and API related activity.

  • Grey button shows activities which are participating in flow.
  • Blue button shows all the methods/API/DB calls involved during whole process
  • Red button shows also those methos/API/DB calls were parsed from log but not taken into consideration due to configured reasons.
